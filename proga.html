1. Фундамент и архитектура

Шаг 1 
Создай основу проекта на Godot 4 C# для кроссплатформенности.

Строго соблюдай принципы SOLID:
- S: один класс — одна ответственность
- O: расширение через интерфейсы, не изменение
- L: наследники корректно заменяют родителей
- I: много маленьких интерфейсов
- D: зависимости от абстракций, внедрение через конструктор/DI

Реализуй центральные интерфейсы и простой DI-контейнер (ручной Service Locator или через конструкторы). Структура папок: Core, Gameplay, Network, UI, Data.

Сделай интерфейсы:
- IInputProvider (получение ввода)
- IGameState (текущее состояние игры)
- IModLoader (загрузка/перезагрузка .txt конфигов)
- INetworkService (абстракция над LiteNetLib)
- IAssetProvider (загрузка спрайтов/звуков)

Создай пустые заглушки для этих сервисов (StubInputProvider, StubGameState и т.д.), чтобы проект компилировался и был готов к наполнению.

Цель: получить компилируемый проект с внедрёнными зависимостями, строго соответствующий SOLID, и готовой архитектурой для следующих шагов.

### Требования SOLID (строго обязательно)
- S: Каждый публичный класс должен иметь ровно одну зону ответственности (можно описать в одном предложении без "и").
- O: Новое поведение добавляется только через реализацию интерфейсов, без изменения существующих классов.
- L: Любой класс, реализующий интерфейс, должен бытьusable вместо любого другого класса с тем же интерфейсом без изменения корректности программы.
- I: Интерфейсы содержат не более 3-5 методов. Разделяй толстые интерфейсы на несколько маленьких.
- D: Все зависимости внедряются через конструктор. Никаких статических сервис-локаторов (кроме точки входа DI).

2. Парсер .txt и горячая перезагрузка

Шаг 2

7. Сеть (P2P хост-клиент)

Шаг 7 
Интегрируй LiteNetLib через INetworkService.
- Хост: запуск сервера на указанном порту, загрузка текущих .txt конфигов
- Клиент: подключение по IP:порт
- При подключении клиента хост автоматически отправляет все файлы из папки mods/ и необходимые ассеты (через IAssetProvider)
- Синхронизация: хост авторитарно управляет игровым состоянием, клиенты шлют только ввод
- Команда /reload на хосте вызывает перезагрузку модов и рассылает обновлённые файлы всем клиентам
- Сетевые сообщения строго типизированы, свои классы для каждого типа

Соблюдай SOLID: INetworkService абстрагирует конкретную библиотеку. Сообщения — отдельные классы с общим интерфейсом INetworkMessage. Логика сети не смешана с игровой логикой.
