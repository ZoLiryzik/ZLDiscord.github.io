ZLDiscord v1.4 — Документация

Введение

ZLDiscord — универсальный плагин для Paper 1.16.5–1.21.x (и выше), интегрирующий Minecraft с Discord. Встроенный шаблонизатор с функциями-чейнами, HTTP LongPoll для внешних скриптов, кастомные команды с двойным scope (Minecraft + Discord), голосования с таймером, DataStore, модалки, реакции, SelectMenu, кастомные PAPI-плейсхолдеры, Discod-роли и многое другое.

JDA 5.x Java 17+ Paper 1.16–1.21+ MySQL / YAML

Установка

  1. Скопировать ZoLiryzik.jar в plugins/
  2. Запустить LongPoll-бота: node longpoll-server.js &
  3. Перезапустить сервер (stop → start, НЕ /reload)
  4. Настроить config.yml: токен бота, Guild ID, роли
  5. Подключить LongPoll: /zlongpoll create demo http://localhost:17523/poll secret 25
  6. Настроить дополнительные YAML (modals.yml, reactions.yml, placeholders.yml) по желанию

Команды

/zevent — Голосование

Minecraft:

/zevent "Название" "команда" "время" channel_id [true|false]

Discord (slash-команда):

/zevent title: command: duration: channel: [route:] [resolve-now:]

Создаёт Discord-голосование ✅/❌ с таймером. При победе ✅ выполняется консольная команда.

Параметр Описание
title / Название Заголовок embed (поддерживает %placeholder%)
command / команда Minecraft-команда консоли при победе ✅
duration / время Длительность: 5s, 10m, 1h
channel / channel_id Текстовый канал Discord
route Целевой сервер сети (например survival, creative)
resolve-now [true|false] Резолвить PAPI плейсхолдеры сразу (true) или только при выполнении команды (false)

После таймера подсчитываются ✅ / ❌. При победе ✅ выполняется команда. Discord-версия также поддерживает route и resolve-now как опциональные параметры.

/zlink / /zunlink — Привязка Discord

/zlink — получить код в Minecraft
/zunlink — отвязать в Discord (ЛС бота)

/zembed — Embed

/zembed "имя" "заголовок" "описание" "channel_id" [картинка] [миниатюра] [футер] [иконка_футера]

PAPI

/zembedmanage — Управление embed

/zembedmanage delete <имя>
/zembedmanage edit_title <имя> "новый заголовок"
/zembedmanage edit_description <имя> "новое описание"

/zcc — Кастомные команды

/zcc list             — список команд
/zcc info <name>      — инфо (шаблон, кулдаун)
/zcc run <name>       — выполнить
/zcc create <name>    — создать
/zcc remove <name>    — удалить
/zcc reload           — перезагрузить

/zattr — DataStore

/zattr get <bot|member|player> <key> [player]
/zattr set <bot|member|player> <key> <value> [player]
/zattr inc|dec <...> <key> <amount> [player]
/zattr list <bot|member|player> [player]

/zlongpoll — LongPoll

/zlongpoll create <name> <url> <key> [wait]
/zlongpoll list
/zlongpoll remove <name>
/zlongpoll reload

Управление Discord-ролями NEW v1.4

/rolegive <member> <role>       — выдать роль
/roleremove <member> <role>     — снять роль
/rolemenu                        — меню выбора ролей
/role_toggle [member] [role]     — переключить роль

Команды в Discord

Префикс текстовых команд: z! (меняется в config.yml → discord-command-prefix).

  • /zevent — голосование (роли embed-creator)
  • /link <code> — привязать (ЛС)
  • /unlink — отвязать (ЛС)
  • Slash-команды из custom_commands.yml — авторегистрация
  • Текстовые через префикс: z!warn, z!report

Кастомные команды

Файл custom_commands.yml:

commands:
  hello:
    file: "commands/hello.yml"
    description: "Приветствие"
    cooldown: 10
    enabled: true
    scope: all              # all / minecraft / discord
    permission: zoliryzik.hello
    permission-message: "&cНет прав"
    command_mc: hello
    command_ds: hello
    aliases_mc: ["h"]
    aliases_ds: ["h"]
    discord-roles: ["role_id_1", "role_id_2"]

Вложенность папок: commands/lp/lp.yml

Система шаблонов

Мощный шаблонизатор с синтаксисом <~ ~>. Работает в кастомных командах, embed, LongPoll-скриптах.

Переменные

Переменная Где Описание
args всегда List аргументов
_args_raw всегда Строка аргументов
_cmd_name всегда Имя команды
_lang всегда Язык: ru/en
mc_online всегда Онлайн
mc_max_players всегда Макс. игроков
mc_tps всегда TPS
mc_motd всегда MOTD
_player_name Minecraft Имя
_player_uuid Minecraft UUID
_player_world Minecraft Мир
_discord_member_id Discord ID участника
_discord_member_name Discord Имя
_discord_member_nick Discord Никнейм

Теги

Тег Описание
<~ expr ~> Вывести значение
<set var = expr> Присвоить
<if>...<elseif>...<else>...</if> Условие
<for var in list>...</for> Цикл
<do action(...)> Действие
<require cond return "msg"> Проверка
<return "msg"> Выход
<global var = expr> Глобальная
<# comment #> Комментарий

Функции

Строки: lower, upper, capitalize, trim, title, replace(a,b,s), split(sep,s), length, format, after(N)

Проверки: defined, empty, not, number, iterable, even, odd, contains, startsWith, endsWith, typeof

Математика: abs, round, number_format, random(min,max), random_hex, random_uuid

Коллекции: first, last, reverse, slice, sort, shuffle, batch, concat, join, keys, values, default

PAPI: papi("placeholder" [, player])

DataStore: getattribute(key), member.getattribute(key), player.getattribute(key), bot.getattribute(key), find_player_by_session(id)

HTTP: http_get(url), http_post(url, body|k,v...), http_put, http_delete, http_patch, http_code(resp), http_body(resp), http_header(n,v), http_bearer(tok), json_parse(str)

Действия (<do>)

Модальные окна NEW v1.4

Файл modals.yml — настройка Discord Modal UI-компонентов, вызываемых из Minecraft или Discord команд.

modals:
  report:
    title: "Отправить репорт"
    command: "zreport {player} {reason}"
    fields:
      - label: "Ник игрока"
        style: "short"         # short / paragraph
        placeholder: "Введите ник"
        required: true
        min_length: 1
        max_length: 32
        default: ""
      - label: "Причина"
        style: "paragraph"
        placeholder: "Опишите причину"
        required: true
        min_length: 10
        max_length: 500

Параметры поля:

Параметр Описание
label Заголовок поля
style short (одна строка) / paragraph (много строк)
placeholder Подсказка
required true / false
min_length / max_length Мин./макс. длина
default Значение по умолчанию

В команде command используются плейсхолдеры {field_label}, передающие значения из модалки.

Реакции NEW v1.4

Файл reactions.yml — настройка автоматических реакций на сообщения.

reactions:
  # Реакция-триггер: игрок ставит реакцию → получает роль
  welcome:
    type: "reaction_add"      # reaction_add / reaction_remove
    channel_id: "000000000000000000"
    message_id: "000000000000000000"    # ID сообщения (или "any")
    emoji: "✅"
    role_id: "000000000000000000"       # ID выдаваемой/снимаемой роли
    remove_on_unreact: true             # снять роль при снятии реакции

  # Реакция-триггер на любое сообщение
  upvote:
    type: "reaction_add"
    channel_id: "any"
    message_id: "any"
    emoji: "⭐"
    command: "mc.command('say '+_discord_member_name+' поставил звезду')"

Параметры:

Параметр Описание
type reaction_add (добавление) / reaction_remove (удаление)
channel_id ID канала (или "any")
message_id ID сообщения (или "any")
emoji Эмодзи (Unicode или ID кастомного)
role_id Роль для выдачи/снятия (опционально)
remove_on_unreact Снимать роль при отмене реакции
command Команда шаблонизатора при срабатывании (опционально)

SelectMenu NEW v1.4

Discord SelectMenu (выпадающий список), настраиваемый в кастомных командах через <do>.

<do discord.embed(_discord_channel_id)
  .withTitle("Выбери роль")
  .withDescription("Нажми на кнопку или выбери из списка")
  .withColor("#5865F2")
  .withSelectMenu({
    "custom_id": "role_select",
    "placeholder": "Выбери роль...",
    "min_values": 1,
    "max_values": 1,
    "options": [
      {"label": "Админ", "value": "role_admin", "description": "Админ", "emoji": "👑"},
      {"label": "Модератор", "value": "role_mod", "description": "Модер", "emoji": "🛡"},
      {"label": "Игрок", "value": "role_player", "description": "Обычный игрок", "emoji": "🎮"}
    ]
  })
  .send()>

Обработка выбора настраивается в custom_commands.yml через Discord-команды, реагирующие на custom_id.

PlaceholderAPI NEW v1.4

Кастомные плейсхолдеры — файл placeholders.yml:

placeholders:
  # Статический плейсхолдер
  server_type:
    description: "Тип сервера"
    value: "Выживание"

  # Динамический — запускает шаблон
  online_info:
    description: "Информация об онлайне"
    template: "&a❤ &f<~ mc_online ~>/<~ mc_max_players ~>"

  # С привязкой к игроку
  player_guild_name:
    description: "Название гильдии игрока"
    template: "<~ papi('%some_plugin_guild%', _player_name) ~>"
    requires_player: true   # true — подставляет игрока из контекста

Хранилище

Поддерживается два типа: YAML (файлы на диске) и MySQL (реляционная БД). Настройка в config.yml → storage.

Таблица Назначение
zldiscord_links Привязка Discord-аккаунтов к Minecraft
zldiscord_linking_sessions Сессии привязки (код подтверждения)
zldiscord_event_tracker Голосования (/zevent)
zldiscord_templates Сохранённые embed-шаблоны
zldiscord_cross_commands Очередь кросс-серверных команд
zldiscord_* Префикс таблиц (меняется в config)

MySQL: HikariCP + MySQL Connector 8.x (встроен в JAR, не требует внешней установки).

FAQ / Частые вопросы

Плагин не запускается, пишет «NoClassDefFoundError»

Убедитесь, что используете Java 17+. Проверьте java -version.

Discord бот не подключается

Проверьте токен в config.yml или переменной окружения DISCORD_TOKEN. Убедитесь, что бот имеет намерения (Intents): Server Members, Message Content, Guild Messages, Message Reactions.

LongPoll не отвечает

Запущен ли Node.js-бот? Проверьте ps aux | grep longpoll. Совпадает ли порт и ключ?
Проверка: curl http://localhost:17523/poll -H "Authorization: Bearer secret"

Как обновить с v1.3 на v1.4?

  1. stop сервер
  2. Заменить ZoLiryzik.jar
  3. start сервер
  4. Проверить config.yml (появилось поле debug)
  5. Новые YAML (modals.yml, reactions.yml, placeholders.yml) создадутся автоматически с примерами

Не работают кастомные плейсхолдеры

Убедитесь, что PlaceholderAPI установлен. Проверьте, что в placeholders.yml нет синтаксических ошибок. Перезагрузите: /zcc reload.

Поддерживается ли Folia?

Да. Плагин корректно работает на Folia (регионы, асинхронные задачи).

Как перенести данные с YAML на MySQL?

  1. Создать БД и пользователя MySQL
  2. Настроить config.yml → storage → mysql
  3. Поменять storage → type на "mysql"
  4. Перезапустить сервер — данные скопируются автоматически
