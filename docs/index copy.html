🇷🇺 Документация

1. Команды

1.1 /zevent

Создать голосование в Discord. Через заданное время подводит итоги (✅ / ❌) и выполняет команду при победе ✅.

/zevent "Название" "команда" "время" "channel_id"
ПараметрОписание
НазваниеЗаголовок embed. Поддерживает %placeholder%
командаMinecraft-команда при победе ✅ (консоль)
время5s, 10m, 1h — длительность
channel_idID текстового канала Discord

1.2 /zcc — Кастомные команды

/zcc list             — список
/zcc run <name>       — выполнить
/zcc create <name>    — создать (\n для новой строки)
/zcc remove <name>    — удалить
/zcc reload           — перезагрузить

Шаблоны в отдельных файлах commands/... и подпапках:

  lp:
    template: commands/lp/lp.yml
    description: Отправить событие в LongPoll
    cooldown: 3
    enabled: true

1.3 /zlink / /zunlink

Привязка/отвязка Discord-аккаунта. Только для игроков.

1.4 /zembed

/zembed "имя" "заголовок" "описание" "channel_id" [картинка] [миниатюра] [футер] [иконка_футера]

Создаёт embed в Discord, сохраняет messageId. PAPI

1.5 /zembedmanage

/zembedmanage delete <имя>
/zembedmanage edit_title <имя> "новый заголовок"
/zembedmanage edit_description <имя> "новое описание"

1.6 /zattr — DataStore

/zattr get <bot|member|player> <key> [player]
/zattr set <bot|member|player> <key> <value> [player]
/zattr inc|dec <...> <key> <amount> [player]
/zattr list <bot|member|player> [player]

1.7 /zlongpoll

/zlongpoll create <name> <url> <key> [wait]
/zlongpoll list
/zlongpoll remove <name>
/zlongpoll reload

2.3 Функции

Проверки: defined, empty, not, number, iterable, even, odd, contains, startsWith, endsWith, typeof

Важно http_post с чётным кол-вом пар сам строит JSON:

3. LongPoll Bot API

Node.js-бот на http://localhost:17523. Аутентификация: Authorization: Bearer secret.

МетодEndpointТело
POST/event{"type":"...", ...}
POST/api/config{"key":"value"}
GET/api/script?type=X—
POST/api/script?type=X{"script":"..."}

Конфигурация подключения

Каждое подключение — отдельный YAML-файл в longpoll/:

ПолеОписание
serverURL LongPoll сервера
keyBearer токен
waitСекунд ожидания (0 = бесконечно, макс 60)
enabledtrue/false — подключение активно? (по умолч. true)
debugtrue/false — лог событий
scriptИмя папки в papca/ или код шаблона

Автоперезапуск При любой ошибке подключение ждёт 5 сек и переподключается автоматически. Работает 24/7 пока enabled: true.

LongPoll скрипты

longpoll/papca/<connection>/<type>.yml

ПеременнаяОписание
_event.typeТип события
_event.player, _event.amount, ...Поля из JSON

Важно Только <do action(...)>, НЕ <action(...)>.

4. Установка

  1. Скопировать ZoLiryzik.jar в plugins/
  2. Запустить бота: node longpoll-server.js &
  3. Перезапустить сервер (stop → start, не /reload)
  4. /zlongpoll create demo http://localhost:17523/poll secret 25

config.yml save-examples: false — отключает авто-создание примеров команд и LongPoll скриптов при старте.

5. Переменные окружения

ПеременнаяОписание
DISCORD_TOKENТокен Discord бота
LONGPOLL_KEYКлюч API (умолч. secret)
LONGPOLL_PORTПорт (умолч. 17523)

6. PlaceholderAPI

Плагин регистрирует расширение zoliryzik:

ПлейсхолдерОписание
%zoliryzik_in_vc%Проверяет, находится ли привязанный Discord-участник в голосовом канале. Возвращает сообщение из config.yml (placeholder-messages.*)
%zoliryzik_attr_member_<key>%Атрибут DataStore для привязанного Discord-участника
%zoliryzik_attr_player_<key>%Атрибут DataStore для игрока
%zoliryzik_attr_bot_<key>%Атрибут DataStore для бота

Также работает любой внешний %placeholder% через PlaceholderAPI.setPlaceholders():

  • Игрок — от имени отправителя
  • Консоль — от первого онлайн
  • papi("x", "Nick") — для любого игрока

7. Примеры

LP команда из майнкрафта

commands/lp/lp.yml:
<set _ = http_header("Authorization", "Bearer secret")>
<set _type = args[0]>
<set _resp = http_post("http://localhost:17523/event",
  "type", _type, "player", _player_name, "world", _player_world,
  "cmd", _cmd_name, "args", _args_raw)>
<do mc.broadcast("&a[LP] &f" + _type + " &a✓")>

Donate-уведомление (с _lang)

<if _event.amount >= 5000>
  <if _lang == "ru">
    <do mc.broadcast("&a&l\u2B50 Мега-донат от &e" + _event.nickname)>
  <else>
    <do mc.broadcast("&a&l\u2B50 Mega-donate from &e" + _event.nickname)>
  </if>
<else>
  <if _lang == "ru">
    <do mc.broadcast("&a❤ " + _event.nickname + " задонатил &e" + _event.amount)>
  <else>
    <do mc.broadcast("&a❤ " + _event.nickname + " donated &e" + _event.amount)>
  </if>
</if>

PAPI для другого игрока

<set target = args[0]>
<~ papi("player_health", target) ~>
<~ papi("player_world", target) ~>

Create a Discord vote. After the timer runs out, counts ✅ / ❌ reactions. If ✅ wins, executes the command.

/zevent "title" "command" "duration" "channel_id"
ParamDescription
titleEmbed title. Supports %placeholder%
commandMinecraft command on ✅ win (console)
duration5s, 10m, 1h
channel_idDiscord text channel ID

3. LongPoll Bot API

Node.js bot at http://localhost:17523. Auth: Authorization: Bearer secret.

MethodEndpointBody
POST/event{"type":"...", ...}
POST/api/config{"key":"value"}
GET/api/script?type=X—
POST/api/script?type=X{"script":"..."}

Connection Config

Each connection is a separate YAML file in longpoll/:

FieldDescription
serverLongPoll server URL
keyBearer token
waitPoll wait seconds (0 = infinite, max 60)
enabledtrue/false — is connection active? (default true)
debugtrue/false — log events
scriptFolder name in papca/ or inline template code

Auto-reconnect On any error the connection waits 5 seconds then reconnects automatically. Runs 24/7 while enabled: true.

LongPoll Scripts

longpoll/papca/<connection>/<type>.yml

VariableDescription
_event.typeEvent type
_event.player, _event.amount, ...Fields from JSON

Important Use <do action(...)>, NOT <action(...)>.

6. PlaceholderAPI

The plugin registers the zoliryzik expansion:

PlaceholderDescription
%zoliryzik_in_vc%Checks if the linked Discord member is in a voice channel. Returns message from config.yml (placeholder-messages.*)
%zoliryzik_attr_member_<key>%DataStore attribute for linked Discord member
%zoliryzik_attr_player_<key>%DataStore attribute for the player
%zoliryzik_attr_bot_<key>%DataStore attribute for the bot

Also resolves any external %placeholder% via PlaceholderAPI.setPlaceholders():

  • Player — resolves for command sender
  • Console — resolves for first online player
  • papi("x", "Nick") — target any player by name
