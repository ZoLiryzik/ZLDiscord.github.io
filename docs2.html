ZLDiscord v1.2

Minecraft ↔ Discord интеграция: кастомные команды с раздельными именами для MC и Discord

Введение

ZLDiscord — Paper-плагин для Minecraft 1.16.5+ (Java 11+). Позволяет создавать кастомные команды, которые работают в майнкрафте, в дискорде или сразу в обеих средах.

  • Кастомные команды с шаблонизатором (file: внешние .yml)
  • Раздельные имена для MC и Discord: command_mc, command_ds, aliases_mc, aliases_ds
  • Разграничение по среде: scope: all / minecraft / discord
  • @упоминания в Discord → ID и имя в шаблоне
  • Discord embed с полями, кнопками, реакциями
  • Привязка аккаунтов + PlaceholderAPI

Установка

  1. Скачать ZoLiryzik.jar
  2. Положить в plugins/
  3. Запустить сервер — создадутся plugins/ZLDiscord/config.yml, custom_commands.yml, папка commands/ с примерами
  4. Настроить bot token, guild ID в config.yml
  5. Перезапустить сервер
Важно: Java 11+. В Discord Developer Portal включить Intents: GUILD_MESSAGES, MESSAGE_CONTENT, DIRECT_MESSAGES, GUILD_MEMBERS.

Кастомные команды

Главное

  • Команды описываются в plugins/ZLDiscord/custom_commands.yml
  • Шаблоны — внешние .yml файлы в commands/, подключаются через file:
  • Ключ YAML — внутренний ID. Имена в MC и Discord задаются отдельно
  • Для перезагрузки: /zcc reload

Синтаксис custom_commands.yml

commands:
  # Команда для всех: /hello в MC, z!hello в Discord
  hello:
    file: "commands/hello.yml"
    description: "Приветствие"
    cooldown: 10
    aliases: ["hi", "privet"]
    enabled: true
    scope: all

  # Только MC
  online:
    file: "commands/online.yml"
    description: "Онлайн сервера"
    cooldown: 5
    enabled: true
    scope: minecraft

  # Только Discord, command_ds = другое имя
  warn:
    file: "commands/warn.yml"
    description: "Репорт на @пользователя"
    command_ds: warns         # z!warns в Discord
    cooldown: 30
    aliases_ds: ["warn"]      # z!warn тоже работает
    enabled: true
    scope: discord

  # Только MC, command_mc = другое имя
  heal:
    file: "commands/heal.yml"
    description: "Вылечить игрока"
    command_mc: heal          # /heal в MC
    cooldown: 10
    enabled: true
    scope: minecraft

  # Везде, раздельные алиасы
  info:
    file: "commands/info.yml"
    description: "Информация об игроке"
    command_mc: info
    command_ds: info
    cooldown: 3
    aliases_mc: ["i"]         # /i в MC
    aliases_ds: ["i"]         # z!i в Discord
    enabled: true
    scope: all

Как вызывать

Управление

  • /zcc list — список команд
  • /zcc run <ключ> [args] — выполнить
  • /zcc create <имя> <шаблон> — создать
  • /zcc remove <имя> — удалить
  • /zcc reload — перезагрузить из файла

Те же команды работают в Discord через slash: /zcc list, /zcc run и т.д.

Шаблонизатор

Шаблоны пишутся без YAML-обёртки — чистый текст в plugins/ZLDiscord/commands/*.yml.

DataStore — атрибуты (persistent data)

Плагин сохраняет произвольные атрибуты в YAML-файлы. Атрибуты сохраняются между перезапусками сервера.

ФайлНазначение
bot_attributes.ymlГлобальные атрибуты бота (доступны из любой команды)
member_attributes.ymlАтрибуты участников Discord, ключ = discordUserId.ключ

Выражения (получение значения)

СинтаксисОписание
<~ member.getAttribute("ключ") ~>Получить атрибут текущего участника Discord
<~ bot.getAttribute("ключ") ~>Получить глобальный атрибут бота
<set x = member.getAttribute("coins")>Присвоить переменной
<if member.getAttribute("warns") >= 3>Использовать в условиях

Действия (изменение)

СинтаксисОписание
<do member.getAttribute("ключ").update("значение")>Установить значение (строка/число)
<do member.getAttribute("монеты").increment(10)>Увеличить число
<do member.getAttribute("монеты").decrement(5)>Уменьшить число
<do bot.getAttribute("глобал").update(аргумент + "_suffix")>С конкатенацией
<do bot.getAttribute("counter").increment(1)>Инкремент глобального счётчика

Важно: member всегда относится к участнику, вызвавшему команду. Если команда из Minecraft, member.getAttribute() вернёт null.
